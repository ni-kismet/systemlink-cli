Fleet summary, calibration history, location tracking, and full CRUD with bus-type and model filtering.

Systems & Jobs

Monitor connected systems, manage packages & feeds, track jobs, and generate software/hardware reports.

WebApp Lifecycle

📁

Files & Notebooks

Upload, download, query files. Manage Jupyter notebooks with workspace filtering and interface assignment.

INSTALL

Up and running in seconds

macOS / Linux

Requires Homebrew

brew tap ni-kismet/homebrew-ni
brew install slcli

Windows

Requires Scoop

scoop bucket add ni-kismet https://github.com/ni-kismet/scoop-ni
scoop install slcli

Standalone Binary

No package manager needed

# macOS
curl -fsSL https://github.com/ni-kismet/systemlink-cli/releases/latest/download/slcli-macos.tar.gz | tar xz
sudo mv slcli/slcli /usr/local/bin/

# Linux
curl -fsSL https://github.com/ni-kismet/systemlink-cli/releases/latest/download/slcli-linux.tar.gz | tar xz
sudo mv slcli/slcli /usr/local/bin/

pip (PyPI)

Requires Python 3.11+

pip install slcli

WORKFLOWS

Build with AI, deploy with confidence

Combine the CLI with AI agent skills to build, test, and deploy SystemLink web applications and demo environments.

1

Install AI Skills

Install the systemlink-webapp skill to give your AI assistant full knowledge of Nimble Angular, SystemLink TypeScript clients, and deployment patterns.

slcli skill install --client all
2

Build Nimble Angular WebApps

Scaffold a project with the Angular template, then ask your AI to build it using @ni/nimble-angular and @ni/systemlink-clients-ts.

slcli webapp init --template angular \
  --directory ./fleet-dashboard
# AI builds it using the skill
slcli webapp publish ./fleet-dashboard/dist \
  --name "Fleet Dashboard" --workspace Prod
3

Provision Demo Environments

Install curated example datasets for training, demos, and evaluation — complete with systems, assets, and test data.

slcli example list
slcli example install demo-complete-workflow \
  --workspace Training
